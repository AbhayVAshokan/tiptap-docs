When exporting to `Docx`, you can define custom styles that will be applied to the exported document. This is useful when you want to have a consistent look and feel across your documents.

```ts
editor
    .chain()
    .exportDocx(
    (result) => {
        // Your export logic should go here
    },
    {
        exportType: 'blob',
    },
    [], // Custom nodes
    {
        paragraphStyles: [
        // Heading 1 style override
        {
            id: 'Heading1',
            name: 'Heading 1',
            basedOn: 'Normal',
            next: 'Normal',
            quickFormat: true,
            // How the text looks for this particular paragraph style
            run: {
                font: 'Aptos Light',
                size: pointsToHalfPoints(16),
                bold: true,
                color: 'FF0000',
            },
            // Styles applied to the paragraph element itself
            paragraph: {
                spacing: {
                    before: pointsToTwips(12),
                    after: pointsToTwips(6),
                    line: lineHeightToDocx(1.15),
                },
            },
        },
        ],
    }, // Style overrides
    )
    .run()
```

In the example above ☝️ we are exporting a document with a custom `Heading 1` style. The style is based on the `Normal` style, has a red color, and uses the `Aptos Light` font. The paragraph spacing is set to `12pt` before and `6pt` after the paragraph. The line height is set to `1.15`.

You can also create custom styling for other elements like `Heading 2`, `Heading 3`, `List Bullet`, `List Number`, etc. The `paragraphStyles` array accepts an array of objects with the following properties:

- `id`: The unique identifier of the style
- `name`: The name of the style
- `basedOn`: The style that this style is based on
- `next`: The style that should be applied after this style
- `quickFormat`: Whether the style should be available in the quick format menu
- `run`: The style for the text run
- `paragraph`: The style for the paragraph

The `run` object accepts the following properties:

- `font`: The font family to be used for the text
- `size`: The font size in half points (multiply your desired point size by 2)
- `bold`: Makes the text bold when set to `true`
- `italics`: Makes the text italic when set to `true`
- `underline`: Applies underline to text. Can specify `color` and `type` (e.g., single, double, thick)
  - `color`: The underline color in hex format (e.g., `FF0000` for red, notice that we don't use `#`)
  - `type`: The underline type (single, double, thick)
- `effect`: Special text effects that can be applied
- `emphasisMark`: Emphasis marks that appear above or below text
- `color`: The text color in hex format (e.g., `FF0000` for red, notice that we don't use `#`)
- `kern`: Controls the spacing between characters in points
- `position`: Controls text position (raised or lowered)
- `smallCaps`: Displays text in small capitals when set to `true`
- `allCaps`: Displays text in all capitals when set to `true`
- `strike`: Applies a single strikethrough to text when set to `true`
- `doubleStrike`: Applies a double strikethrough to text when set to `true`
- `subScript`: Makes text appear smaller and below the baseline
- `superScript`: Makes text appear smaller and above the baseline
- `highlight`: Adds highlighting to text with predefined colors
- `characterSpacing`: Controls the spacing between characters
- `shading`: Applies background shading to text
- `scale`: Adjusts the width of the text (percentage value)

For more advanced styling options and detailed usage you can refer to the `IRunStylePropertiesOptions` type exposed from our package, or refer to the [docx documentation](https://docx.js.org/#/usage/styling-with-js).

The `paragraph` object accepts the following properties:

- `border`: Defines the borders around the paragraph (top, bottom, left, right)
- `shading`: Applies background shading to the entire paragraph
- `numbering`: Controls paragraph numbering settings
  - `reference`: The numbering style reference ID
  - `level`: The level in the numbering hierarchy (0-based)
  - `instance`: Optional instance number for the numbering
  - `custom`: Whether to use custom numbering format
- `alignment`: Sets paragraph alignment (left, center, right, justify)
- `thematicBreak`: Adds a horizontal line break when set to `true`
- `contextualSpacing`: Reduces spacing between paragraphs of the same style
- `rightTabStop`: Sets the position of the right tab stop in twips
- `leftTabStop`: Sets the position of the left tab stop in twips
- `indent`: Controls paragraph indentation
  - Can set first line, hanging, left, and right indents
- `spacing`: Controls paragraph spacing
  - `before`: Space before the paragraph
  - `after`: Space after the paragraph
  - `line`: Line spacing within the paragraph
  - `lineRule`: How line spacing is calculated
- `keepNext`: Keeps this paragraph on the same page as the next paragraph
- `keepLines`: Keeps all lines of the paragraph together on the same page
- `outlineLevel`: Sets the outline level for document organization (1-9)

For more advanced styling options and detailed usage you can refer to the `IParagraphStylePropertiesOptions` type exposed from our package, or refer to the [docx documentation](https://docx.js.org/#/usage/styling-with-js).

## Tiptap's export default styles

Tiptap offers a sensible default styling for the exported document, but you can override these styles by providing your own custom styles. This allows you to create a consistent look and feel across your documents. 

```ts
{
  paragraphStyles: [
    // Normal style (default for most paragraphs)
    {
      id: 'Normal',
      name: 'Normal',
      run: {
        font: 'Aptos',
        size: pointsToHalfPoints(11),
      },
      paragraph: {
        spacing: {
          before: 0,
          after: pointsToTwips(10),
          line: lineHeightToDocx(1.15),
        },
      },
    },
    // List Paragraph style (used for bullets and numbering)
    {
      id: 'ListParagraph',
      name: 'List Paragraph',
      basedOn: 'Normal',
      quickFormat: true,
      run: {
        font: 'Aptos',
        size: pointsToHalfPoints(11),
      },
      paragraph: {
        spacing: {
          before: 0,
          after: pointsToTwips(2),
          line: lineHeightToDocx(1),
        },
      },
    },
    // Heading 1 style
    {
      id: 'Heading1',
      name: 'Heading 1',
      basedOn: 'Normal',
      next: 'Normal',

        font: 'Aptos Light',
```
