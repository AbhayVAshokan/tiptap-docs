The Import/Export conversion service is also accessible via a REST API. This allows you to convert documents on your backend or from outside a Tiptap editor context.

You can make HTTP requests to Tiptap’s conversion endpoints to import or export documents to Tiptap JSON.

## Import API endpoint
`POST /import`

The `/import` endpoint enables the conversion of `docx`, `odt`, or `markdown` files into Tiptap’s JSON format. POST documents to this endpoint and use various parameters to customize how different document elements are handled during the conversion process.

### Example (cURL)
```bash
curl -X POST "https://api.tiptap.dev/v1/import?paragraph=textBlock" \
  -H "Authorization: Bearer <your-jwt-token>" \
  -H "X-App-Id: <your-app-id>" \
  -F "file=@/path/to/document.docx"
```
In this example, the request uploads a DOCX file and specifies that paragraphs should be mapped to a node type textBlock instead of the default paragraph.

### Required headers
If you need to generate or refresh your JWT or retrieve your App ID, see [Getting Started – Authenticate]() for full details on obtaining credentials.

| Name            | Description                                                                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `Authorization` | The JWT token to authenticate the request. Example: `Bearer your-jwt-token`                                                                     |
| `X-App-Id`      | The Convert App-ID from the Collaboration settings page: [https://cloud.tiptap.dev/convert-settings](https://cloud.tiptap.dev/convert-settings) |


### Body

| Name                     | Type     | Description                                                                                                                  |
| ------------------------ | -------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `file`                   | `File`   | The file to convert                                                                                                          |
| `imageUploadCallbackUrl` | `string` | The callback endpoint to upload images that were encountered within the uploaded document, [see more info](#image-uploading) |

### Query parameters

You can override the default node/mark types used during import or export by specifying them as query parameters. You only need to provide these if your editor uses custom node/mark names and you want the imported JSON to use those.

For example, if your schema uses a custom node type called textBlock instead of the default paragraph, you can include `?paragraph=textBlock` in the request URL.

You can similarly adjust headings, lists, marks like bold or italic, and more.

| Name             | Default          |
| ---------------- | ---------------- |
| `paragraph`      | `paragraph`      |
| `heading`        | `heading`        |
| `blockquote`     | `blockquote`     |
| `codeblock`      | `codeblock`      |
| `bulletlist`     | `bulletlist`     |
| `orderedlist`    | `orderedlist`    |
| `listitem`       | `listitem`       |
| `hardbreak`      | `hardbreak`      |
| `horizontalrule` | `horizontalrule` |
| `table`          | `table`          |
| `tablecell`      | `tablecell`      |
| `tableheader`    | `tableheader`    |
| `tablerow`       | `tablerow`       |
| `bold`           | `bold`           |
| `italic`         | `italic`         |
| `underline`      | `underline`      |
| `strikethrough`  | `strike`         |
| `link`           | `link`           |
| `code`           | `code`           |
| `image`          | `image`          |

### Response
On success, the service returns a JSON object representing the document content in Tiptap’s format. You can directly set this as the content of a Tiptap editor or process it as needed.

## Export API endpoint
`POST /export`

The `/export` endpoint converts Tiptap’s JSON content into a chosen file format, such as `docx`, `odt`, or `markdown`. POST your Tiptap JSON to this endpoint and customize how different nodes or marks are handled during the conversion.

### Example (cURL)
``` bash
curl -X POST "https://api.tiptap.dev/v1/export?heading=customHeading" \
  -H "Authorization: Bearer <your-jwt-token>" \
  -H "X-App-Id: <your-app-id>" \

I AM MISSING A MORE THOROUGH EXAMPLE
I AM MISSING A MORE THOROUGH EXAMPLE
I AM MISSING A MORE THOROUGH EXAMPLE
I AM MISSING A MORE THOROUGH EXAMPLE
I AM MISSING A MORE THOROUGH EXAMPLE
```
In this example, the request sends Tiptap JSON and specifies that heading nodes should be mapped to a node type customHeading instead of the default heading. The response returns a DOCX file.


### Required headers
If you need to generate or refresh your JWT or retrieve your App ID, see [Getting Started – Authenticate]() for more details on credentials.

| Name            | Description                                                                                                                                     |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `Authorization` | The JWT token to authenticate the request. Example: `Bearer your-jwt-token`                                                                     |
| `X-App-Id`      | The Convert App-ID from the Collaboration settings page: [https://cloud.tiptap.dev/convert-settings](https://cloud.tiptap.dev/convert-settings) |

### Body

| Name      | Type     | Description                                                  |
| --------- | -------- | ------------------------------------------------------------ |
| `content` | `Object` | The document in Tiptap JSON format.|
| `format`  | `string` | The target file format. Supported options include `docx`, `odt`, `markdown`, or `gfm`. |

### Query parameters
You can override the default node/mark types used during export by specifying them as query parameters. Include these only if your Tiptap editor (or custom schema) uses different node/mark names and you want to map them accordingly in the output document.

For example, if your schema uses a custom heading type called customHeading, you can include ?heading=customHeading in the request URL.

<Callout title="Enable GitHub Flavored Markdown" variant="info"> If you export Markdown, you can optionally specify <code>?gfm=1</code> to enable GitHub Flavored Markdown. </Callout>

| Name             | Default          |
| ---------------- | ---------------- |
| `paragraph`      | `paragraph`      |
| `heading`        | `heading`        |
| `blockquote`     | `blockquote`     |
| `codeblock`      | `codeblock`      |
| `bulletlist`     | `bulletlist`     |
| `orderedlist`    | `orderedlist`    |
| `listitem`       | `listitem`       |
| `hardbreak`      | `hardbreak`      |
| `horizontalrule` | `horizontalrule` |
| `table`          | `table`          |
| `tablecell`      | `tablecell`      |
| `tableheader`    | `tableheader`    |
| `tablerow`       | `tablerow`       |
| `bold`           | `bold`           |
| `italic`         | `italic`         |
| `underline`      | `underline`      |
| `strikethrough`  | `strike`         |
| `link`           | `link`           |
| `code`           | `code`           |

### Response
The service will return the converted file. For `DOCX` or `ODT`, this is binary data and for `Markdown`, it returns a plain text .md file. If you call this endpoint from a client (browser), you’ll likely trigger a download. From a server, you’d receive the binary and can then save it or forward it to a user.
