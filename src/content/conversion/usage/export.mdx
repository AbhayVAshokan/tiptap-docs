Exporting allows you to take the content of a Tiptap editor (Tiptap JSON) and generate a file in a chosen document format (DOCX, ODT, or Markdown).
You can use the Export extension within a Tiptap editor to handle this process, or call the REST API directly. Here we describe using the editor extension.

## Set up the Export extension
After installing the `@tiptap-pro/extension-export` package, configure it in your editor with the required credentials:

```jsx
import { Editor } from '@tiptap/core'
import { Export } from '@tiptap-pro/extension-export'

const editor = new Editor({
  // ... other editor options
  extensions: [
    // ... other extensions
    Export.configure({
      // Your Tiptap Cloud Convert App ID
      appId: '<your-app-id>',
      // A JWT for authentication
      token: '<your-jwt>',
    }),
  ],
})
```

- `appId` and `token` are required for the extension to authenticate with the conversion service (as obtained in the [Install](/conversion/getting-started/install) guide).

## Export a document
To export the current editor content, use the `export` command provided by the extension. You need to specify which format to export to. For example, to export as a Word document (DOCX):

```jsx
// Export the editor content as a DOCX file
editor.chain().focus().export({ format: 'docx' }).run()
```

This will send the editor’s JSON content to the conversion service and trigger a download of the resulting `docx` file in the browser. The `focus()` call ensures the editor is focused (which may be needed in some contexts to initiate commands).

You can also choose other formats by changing the format option (`odt` for OpenDocument Text, `md` for Markdown, or `gfm for GitHub Flavored Markdown).

## Customize the export behavior
By default, the Export command will automatically prompt the user to download the generated file (when used in a browser environment). You can intercept the process using an `onExport` callback to handle errors or override how the file is handled:

```jsx
editor.chain().export({
  format: 'docx',
  onExport(context) {
    const { blob, error, download, filename } = context

    // Handle errors in conversion
    if (error) {
      console.error("Export failed:", error)
      showErrorToast({ message: error.message })
      return
    }

    // Custom behavior: e.g., adjust application state or filename
    isLoading = false  // turn off loading indicator

    // You can modify the filename or process the blob here if needed.
    console.log("Exported file blob:", blob, "default filename:", filename)

    // Trigger browser download of the file
    download()  // (only works in browser contexts)
  },
})
```

In this example, we use `onExport` to catch any error, turn off a loading state flag, log some info, and then call the provided `download()` function to initiate the download. The `context.blob` contains the file data (Blob object) and `context.filename` the suggested filename.

## Export command
Adding the Export extension provides an editor command:

`export(options)` – Exports the editor’s content. You must specify the `format` in the options (`docx`, `odt`, `md`, or `gfm`). Optionally, you can pass `onExport` as shown above, or even a specific `content` to export (if you want to export something other than the editor’s current content).

#### Arguments

| Name       | Type          | Default     | Options           | Description                                                                                                                                                                              |
| ---------- | ------------- | ----------- | ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `format`   | `string`      | `undefined` | `docx,odt,md,gfm` | The format you want to export to                                                                                                                                                         |
| `content`  | `JSONContent` | `undefined` | Any Tiptap JSON   | The Tiptap JSON content you want to export                                                                                                                                               |
| `onExport` | `Function`    | `undefined` | `fn(context)`     | A callback used to customize the export behavior. The context argument includes information about the blob, filename, errors and a `download` function to download the document directly |

## Supported formats

- `docx` - Microsoft Word, Google Docs, OpenOffice, LibreOffice and others
- `odt` - OpenDocument Text format used by OpenOffice, LibreOffice and others
- Commonmark `markdown` - Markdown format used by various editors and platforms
- GFM `markdown` - GitHub Flavored Markdown used by GitHub
